Tarefa de Behavior Driven
Development (BDD) - Engenharia de
Software
Esta atividade tem como objetivo familiarizar os estudantes da disciplina de
Engenharia de Software com a prática de Behavior Driven Development (BDD) e sua
aplicação em um projeto SaaS. Os alunos deverão utilizar os conceitos de cenários
BDD para guiar a implementação e o teste de funcionalidades dentro de uma
aplicação existente.

Objetivo da Tarefa
1. Implementar apenas o sad path para a feature “User can add movie by
searching for it in The Movie Database (TMDb)” (1.)
2. Implementar o cenário imperativo “View movie list after adding 2 movies e
ordenar por ordem alfabética de título de filmes” (2.)
3. Converter o cenário (2.) para o modo declarativo.
Implementar o cenário BDD e demonstrar a execução no seu próprio
"RottenPotatoes".

Cenários para o BDD
1. Try to add nonexistent movie (sad path)

Escopo da Entrega
A entrega desta tarefa consiste em dois componentes principais:

1. Implementação dos Cenários BDD: O aluno deve escrever os arquivos de
feature BDD e implementar os step definitions correspondentes na aplicação
RottenPotatoes. O código-fonte implementado deve ser entregue.
2. Gravação do Vídeo Explicativo: O aluno deve gravar um vídeo demonstrando a
execução do cenário BDD na aplicação e, de forma clara e concisa, explicar a
função de cada step definition implementado.
Instruções
1. Abra o SEU RottenPotatoes.
2. Crie o arquivo .feature conforme o roteiro fornecido.
3. Implemente os métodos de step definition necessários para que o cenário BDD
possa ser executado com sucesso na aplicação.
